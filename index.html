4. Key Learnings & Technical Challenges

This section details specific technical challenges and key learnings encountered during the project's development. Click on each card to expand and read more about the solutions.

A key learning was successfully configuring HDFS to use a replication factor of 4 by editing the `hdfs-site.xml` file within a Dockerized environment. This required a meticulous redundancy test, which involved simulating a DataNode failure and observing HDFS's self-healing process to confirm data integrity and high availability. This proved the resilience of the cluster.

During schema design, challenges arose with column length limitations and data type precision. Specifically, `VARCHAR` lengths for unique URLs had to be reduced to `767` to comply with MySQL's index limits, and `DECIMAL` precision for `price_amount` was increased to prevent "out of range" errors. This highlights the importance of careful schema design for data integrity.

The data ingestion script required sophisticated logic to handle the diverse and often messy price data from different sources. I implemented a robust regular expression-based solution to extract clean numerical values from strings containing various currency symbols, separators, and even price ranges (e.g., `17,50bis 17,73`), ensuring data consistency before insertion.
